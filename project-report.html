Growth Opportunities

  • LinkedIn integration — automated LinkedIn outreach would complete the multi-channel strategy (content series already drafted)
  • Network effects — anonymized success data could improve outreach templates over time
  • Team features — multi-user workspaces for recruiting agencies and career coaches
  • Beta launch ready — onboarding wizard + tour + 1,484 tests = confidence to onboard real users

🛠 Things to Add, Fix & Improve

Fixes & Debt

  • Enable RLS in production — currently feature-flagged off, multi-tenant isolation is untested with real traffic
  • Raise CI coverage floor — 70% minimum is too low for a production SaaS; target 85%+
  • Test DB restore procedure — daily backups exist but never validated with a real restore
  • Consolidate duplicate commits — main branch has duplicate pairs from merge issues (e.g. onboarding commits appear twice)
  • Scout agent data sources — NewsAPI alone is limited; needs real job board APIs (LinkedIn, Indeed) for credible discovery
  • Event bus durability — in-process only; crashes lose events. Migrate to Redis Streams or message queue

Features to Add

  • Staging environment — no pre-production testing environment; deploying directly to production
  • LinkedIn API integration — automated LinkedIn outreach to complete multi-channel strategy
  • Calendar integration — sync interviews with Google Calendar / Outlook
  • Conversion tracking — no analytics on landing page (add Plausible, PostHog, or similar)
  • A/B testing on landing page — optimize signup conversion with variant testing
  • Alerting & on-call — no PagerDuty/Opsgenie integration; failures go unnoticed until manual check
  • Blue/green or canary deploys — Railway does rolling updates but no zero-downtime strategy
  • Team workspaces — multi-user collaboration for agencies and career coaches

Quality Improvements

  • Penetration testing — no OWASP ZAP or manual pentest performed; relying on static scanning only
  • Secrets rotation policy — JWT_SECRET, API keys have no rotation schedule
  • Mutation testing — verify test quality, not just quantity (e.g. mutmut or cosmic-ray)
  • Contract tests — no API contract tests between frontend and backend
  • Load test in CI — Locust exists but isn't part of the CI pipeline
  • Auto-scaling — Railway runs single instance; no horizontal scaling for traffic spikes
  • Onboard real beta users — all infrastructure is ready, but zero real users have tested the system
